# 17. 2차 전직 퀘스트 시스템 — 오지 후반 ~ 해적 섬 시련

> **상태**: 설계안 / 미구현
> **선행 문서**: `06-progression.md` (현재는 Lv 100 자동 해금), `08-balance-reference.md`
> **시뮬레이션 근거**: `scripts/first-job-progression-sim.ts` (Lv100 1차 적당 스펙 기준 사막~유령선 무난, 심연 진입 시 직업 균형 붕괴)

## 0. 설계 철학

자동 해금이던 2차 전직을 **시련 퀘스트**로 게이트하는 이유는 두 가지다.

1. **정체성 체험**: 2차 직업의 패시브를 "받기 전에 그 플레이 스타일을 먼저 강제로 겪게 한다." 광전사가 되려면 먼저 저체력 플레이의 맛을 봐야 하고, 어쌔신이 되려면 한 방의 가치를 알아야 한다.
2. **적정 스펙 게이트**: 1차 적당 스펙(유령 세트 + 명전 lv10 + 도감 15pt)으로 통과 가능하도록 설계 — 즉 "Lv 100 도달 + 후반 1차 컨텐츠를 충분히 돌았다"의 자연스러운 인증.

**그라인드 금지 원칙**:
- "X 몹 1000마리 잡기" 같은 단순 누적 X — 시뮬레이션상 이미 사냥 지표가 충분함
- 보스 1~5회 + 짧은 사냥 시퀀스 + **조건 제약**으로 시련의 밀도를 확보
- 시련 1회 평균 소요시간 목표: **현실 시간 30분~2시간**

## 1. 퀘스트 트리거

| 조건 | 값 |
|---|---|
| 레벨 요구 | Lv 100 |
| 1차 → 2차 후보 | 전사: 광전사/방패병 / 도적: 어쌔신/맹독술사 / 마법사: 원소술사 (단일) |
| 시작 NPC | 마을 시련관 (신규 탭 또는 기존 마을 패널 확장) |
| 동시 진행 | 2차 후보 1개만 활성 (다른 후보 포기 시 진행도 초기화) |

## 2. 적용 컨텐츠 범위

시뮬레이션상 1차 적당 스펙으로 무난한 4개 후반 지역만 사용한다.

| 지역 | 보스 | 보스 메커닉 | 매칭 2차 직업 |
|---|---|---|---|
| 사막 | 거대 전갈 | DOT 독 (5턴 dot_pct 0.05) | **맹독술사** (지속 데미지 마스터) |
| 설원 | 서리 거인 | 단발 고화력 강타 | **광전사** (저체력 폭주) |
| 해적 정박지 | 해적 선장 | 2× 강타 4턴 주기 | **방패병** (반사·견딤) |
| 유령선 | 유령 선장 | 정타 (0.7×ATK) 10턴 주기 | **어쌔신** (정밀 처형) |
| 4지역 횡단 | — | — | **원소술사** (사원소 순례) |

## 3. 직업별 퀘스트

### 3.1 광전사 (Berserker) — 「분노의 시련」

> 패시브 미리보기: ATK +45%, 매턴 maxHP 2% 자해
> 핵심 아이덴티티: **자기 목숨을 깎아 화력을 얻는다**

#### 1단계 — 갈증의 약속
- **목표**: 사막 사냥터 5회 연속 파견 (60초/회) 동안 **시작 시 HP 50% 이하** 상태에서 무사망 완료
- **체험 의도**: 저체력 상태의 텐션 학습. 사막 몹은 ATK 50~55라 적당 스펙 전사 DEF 457로는 견디지만, HP 절반 시작은 안전 마진이 줄어든다.
- **검증**: `Character.currentHp ≤ stats.maxHp / 2` 시작, `DispatchResult.diedEarly === false`

#### 2단계 — 부서진 방패
- **목표**: **무기 외 모든 장비 슬롯을 비운 채** 서리 거인 보스 1회 처치
- **체험 의도**: DEF/HP 보너스 없이 순수 ATK로 보스를 깎는 시간 압박. 적당 스펙 전사 무기만 장착 시 DEF ≈ 200대로 떨어지지만 거인 ATK 340은 어떻게든 막을 수 있고, ATK 손실은 ~80 → 평균 8~12턴 클리어 예상.
- **검증**: `equipped.head/body/gloves/boots == null && weapon != null && bossResult.defeated`

#### 3단계 — 광폭의 정수
- **목표**: 서리 거인의 심장 1개 + **현재 HP의 30%를 시련 제단에 봉헌**(영구 소모, 클리어 후 풀 회복)
- **체험 의도**: 자해를 보상으로 환산하는 광전사의 핵심 거래 행위를 의식화.
- **드롭 확률**: 서리 거인 처치 시 100% (이미 데이터에 있음)

#### 보상
- 광전사 직업 해금
- **광기의 인장** (악세서리/수동 효과: 첫 전투 진입 시 HP -30%, ATK +20% 1회) — 광전사 트레이드 강화 시각화
- 스킬 슬롯 +1 (이미 advancedClass 효과)

### 3.2 방패병 (Paladin) — 「수호자의 서약」

> 패시브 미리보기: 피해 -15%, DEF의 40% 반사
> 핵심 아이덴티티: **맞아주고 되돌려준다**

#### 1단계 — 첫 번째 서약: 견디다
- **목표**: 해적 정박지에서 **8회 연속 파견** 무사망 (60초/회). 해적 선장의 2× 강타가 잡몹에는 없으므로 순수 견디기 시험.
- **체험 의도**: DEF 의존 안정 플레이. 적당 스펙 전사 사망률 0%이므로 통과 가능하지만, 단 1회라도 사망 시 카운터 리셋 — 압박감 부여.
- **검증**: 누적 카운터, `diedEarly === true` 시 0으로 초기화

#### 2단계 — 두 번째 서약: 막아내다
- **목표**: 해적 선장 처치 1회, **단 받은 피해 누계 ≥ maxHP**
- **체험 의도**: "피해 적게 받기"가 아니라 "충분히 받고 살아남기" — 반격 패시브의 입력 측을 의식적으로 활용.
- **트릭**: 해적 선장 4턴 주기 2× 강타에 일부러 노출 → 반사 데미지로 처치. 적당 스펙 전사 maxHP 2683, 해적 선장 ATK 340 → 약 8~12회 직격 필요.
- **검증**: `bossResult.defeated && bossResult.damageTaken >= stats.maxHp`

#### 3단계 — 세 번째 서약: 흔들리지 않는다
- **목표**: 해적 선장 처치 시 **maxSingleHit ≥ 자기 ATK × 2** 단 한 번 이상 (반사 데미지가 자기 평타의 2배 이상 나오도록 = DEF 빌드 강요)
- **체험 의도**: 방패병의 "DEF가 곧 ATK"라는 컨셉 시각화.
- **검증**: `bossResult.maxSingleHit >= stats.atk * 2` (반사 데미지가 maxSingleHit에 포함되도록 사전에 로직 정리 필요)

#### 보상
- 방패병 직업 해금
- **수호자의 대형 방패** (특수 무기, weapon 슬롯 — DEF +40, 반사 +10%)
- 도발/반격강화 스킬 자동 학습 (4종 중 2종)

### 3.3 어쌔신 (Assassin) — 「그림자 입문」

> 패시브 미리보기: 치명타 50%, 치명타 ×2.5배
> 핵심 아이덴티티: **단 한 방의 정확성**

#### 1단계 — 그림자 추적
- **목표**: 해적 단검 30개 수집 (해적 잡몹 0.07 드롭률 → 평균 30~40회 파견 ≈ 30~40분 idle)
- **체험 의도**: 잠입·관찰의 시간. 적당 스펙 도적 사망률 0%로 안전하게 누적.
- **검증**: `materials.pirate_dagger >= 30`

#### 2단계 — 소리 없는 처형
- **목표**: 해적 선장을 **5턴 이내** 처치 (선장의 2× 강타 첫 발이 4턴이므로 그 전 단발 처치 강요)
- **체험 의도**: 보스의 위협 사이클을 회피하는 선제 처형 — 어쌔신의 본질.
- **시뮬값**: 적당 스펙 도적 평균 2T 클리어 (시뮬 기준) → 통과 여유 있음. 중간 스펙으로는 평균 3T → 여전히 가능.
- **검증**: `bossResult.defeated && bossResult.totalTurns <= 5`

#### 3단계 — 유령의 목
- **목표**: 유령 선장을 **단일 치명타 1발로 maxHP 30% 이상** 깎는 순간을 1회 이상 발생
- **체험 의도**: 어쌔신 패시브의 본 게임 시연 — 50% 치명 × 2.5배 공식의 위력 체감.
- **시뮬값**: 적당 스펙 도적 ATK 607, 치명 ×2 = 1214, 유령 선장 maxHP 3500 → 단발 34.7% 가능. 실제 평균 2~3T 클리어이므로 단발 30% 데미지가 1회 이상 자연 발생.
- **검증**: `bossResult.defeated && bossResult.maxSingleHit >= boss.hp * 0.3`

#### 보상
- 어쌔신 직업 해금
- **그림자의 칼날** (특수 단검 — ATK +30, 치명 +5%)
- 어둠 관련 스킬 2종 자동 학습

### 3.4 맹독술사 (Venom Master) — 「독의 길」

> 패시브 미리보기: DOT +50%, 독 스택 캡 +2 (5 → 7)
> 핵심 아이덴티티: **시간이 무기, 직접 안 때린다**

#### 1단계 — 독을 마시다
- **목표**: 사막 비단 20개 + 거대 전갈의 발톱 5개 수집
- **체험 의도**: 사막 자체 친화. 비단 0.07, 발톱은 보스 드롭이라 평균 5번 도전.
- **검증**: 자원 누계 체크

#### 2단계 — 독을 거스르다
- **목표**: 거대 전갈 보스 처치 시 **DOT 누계 데미지 ≥ 직접 데미지 × 2**
- **체험 의도**: 도트 의존 플레이의 명시적 학습. 1차 도적의 dot_on_hit 스킬을 의도적으로 끼워 넣어야 통과 가능.
- **검증**: `BossDispatchResult` 확장 필요 (`damageDealtBySource: { direct, dot }`). 신규 필드.

#### 3단계 — 독을 다스리다
- **목표**: 유령 선장을 **매 턴 시작 시 적 독 스택 ≥ 3** 상태로 유지하며 클리어 (전체 턴의 80% 이상 충족)
- **체험 의도**: 스택 관리의 리듬 — 맹독술사가 cap +2 받기 전에 cap 5 안에서 운영하는 능력 검증.
- **검증**: 매 턴 `enemyDotStacks` 스냅샷 누계, 임계 만족 비율 ≥ 0.8.

#### 보상
- 맹독술사 직업 해금
- **거미 비단 독니** (특수 무기 — ATK +25, 명중 시 독 +1 스택)
- DOT 강화 스킬 2종 자동 학습

### 3.5 원소술사 (Elementalist) — 「사원소 순례」

> 패시브 미리보기: 마법 데미지 +60%, 매턴 시작 시 INT × 0.3 자동 마법 공격
> 핵심 아이덴티티: **자동 마법 + 광범위 화력**

#### 1단계 — 불의 속삭임 (사막)
- **목표**: 거대 전갈 보스를 **물리 데미지 0**으로 처치
- **체험 의도**: 마법 의존 빌드 강요. 1차 마법사의 magic 스킬 의존도 검증.
- **시뮬 가능성**: 마법사 INT 401, 50% 방관 패시브로 거대 전갈 MDEF 22 통과 → 매턴 INT 기반 데미지 ~390. 30턴 안에 4500 HP 처치 가능.
- **검증**: `bossResult.physicalDamageDealt === 0` (필드 신규)

#### 2단계 — 얼음의 침묵 (설원)
- **목표**: 서리 거인을 **회피 0회**로 처치 (자기가 한 번도 회피하지 않음 — AGI 의존 거부)
- **체험 의도**: 원소술사는 회피가 아니라 자동 마법 + MDEF로 버틴다는 정체성.
- **검증**: `bossResult.dodgesByPlayer === 0 && defeated`

#### 3단계 — 번개의 충격 (해적 정박지)
- **목표**: 단일 파견(60초)에서 **마법 누계 데미지 ≥ 총 데미지의 70%** 이며 처치 ≥ 25
- **체험 의도**: 오토 마법 패시브 시연 — 매턴 자동 마법으로 잡몹을 쓸어담는 감각.
- **검증**: 신규 필드 + `totalKills >= 25`

#### 4단계 — 영혼의 정수 (유령선)
- **목표**: 유령 선장을 30턴 안에 처치하며 **스킬 발동 ≥ 5회**
- **체험 의도**: 다양한 원소(스킬 슬롯 5개) 활용 검증.
- **검증**: `Object.values(skillActivations).reduce((s,n)=>s+n,0) >= 5 && defeated`

#### 보상
- 원소술사 직업 해금
- **사원소의 지팡이** (특수 무기 — INT +30, 마법 +10%)
- 매 단계 클리어 시 정수 1개씩 → 4정수 모이면 마지막 봉헌 의식

## 4. 공통 시스템 사항

### 4.1 진행도 데이터 모델

```ts
type SecondJobQuest = {
  candidate: AdvancedClassId;       // 선택한 2차 후보
  startedAt: number;
  stage: number;                    // 1~3 (광전사) / 1~4 (원소술사)
  progress: Partial<Record<string, number>>;  // 카운터·재료 누계
  completedStages: number[];
};

type Character = {
  // ...기존 필드
  secondJobQuest?: SecondJobQuest;
};
```

### 4.2 검증 측면 logic.ts 확장 포인트

| 필드 | 위치 | 용도 |
|---|---|---|
| `BossDispatchResult.physicalDamageDealt` | resolveBossDispatch | 원소술사 1단계 |
| `BossDispatchResult.dotDamageDealt` | resolveBossDispatch | 맹독술사 2단계 |
| `BossDispatchResult.turnsWithEnemyDotGte` | resolveBossDispatch | 맹독술사 3단계 |
| `BossDispatchResult.maxSingleHit` (이미 존재) | — | 어쌔신 3단계 |
| `DispatchResult.diedEarly` (이미 존재) | — | 광전사 1단계, 방패병 1단계 |

### 4.3 실패·재도전

- 단계 실패 시 **즉시 재도전 가능**, 페널티 없음 (재화/시간만 소모)
- 후보 포기·전환은 가능하지만 진행도 초기화 (스토리상 "다른 길을 간다"의 무게)

### 4.4 UI 흐름

```
마을 → [시련관] 탭
  ├─ 후보 선택 (1차 직업별 2개 또는 1개)
  ├─ 현재 단계 / 조건 체크리스트
  ├─ 도전 버튼 → 사냥/보스 파견 화면으로 점프
  └─ 보상 수령 (모든 단계 완료 시)
```
