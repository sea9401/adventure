# 컨텐츠 시스템

> 코드: `src/lib/game/data.ts` (REGIONS, COOP_BOSSES), `src/lib/game/arena.ts`, `src/app/ArenaPanel.tsx`, `src/app/CoopBossPanel.tsx`

## 지역 (필드)

11개 지역 + 4개 그룹.

| Group                    | 지역                            | 권장 레벨대 | 일반 적                                                                           |
| ------------------------ | ------------------------------- | ----------- | --------------------------------------------------------------------------------- |
| `outskirts` (마을 외곽)  | 평야 / 외곽 숲 / 동굴           | 1~30        | 슬라임 / 들쥐 / 들개 / 산적 / 동굴 거미 / 수정 정령                               |
| `wilderness` (오지)      | 낡은 유적 / 사막 / 설원         | 30~80       | 돌 가디언 / 유적 망령 / 모래 도적 / 모래 전갈 / 눈 늑대 / 얼음 거인               |
| `pirate_isles` (해적 섬) | 해적 정박지 / 유령선            | 80~130      | 해적 / 갑판원 / 유령 선원 / 해골 검사                                             |
| `abyss` (심연)           | 심연 입구 / 심연 균열 / 심연 핵 | 130~200     | 그림자 정찰병 / 심연 거머리 / 균열의 망령 / 공허 추격자 / 심연의 사도 / 공허 거인 |

각 지역에는:

- `enemies`: 일반 적 2종
- `treasure`: 확률 보물 (gold + materials) — **분당 1회 독립 굴림** (§ 보물 굴림 참고)
- `boss`: 필드 보스 1종 (드랍 + 유니크 드랍)
- `drops`: 골드/철 범위
- `expReward`: 경험치 보상
- `durationMs`: 1턴당 시간

## 탐험 (Dispatch)

지역에서 일반 탐험. 시간을 선택해 출발.

| 시간            | 보상 효율 |
| --------------- | --------- |
| 60초 (1분)      | 100%      |
| 1,800초 (30분)  | 88%       |
| 7,200초 (2시간) | 75%       |

활성 플레이 우대: 짧게 자주 돌릴수록 시간당 보상이 높음(1분 = 100%, 2시간 = 75%).
방치 옵션도 의미 있게 — 격차는 25% 폭으로 제한해 활성/방치 모두 선택 동기 부여.
전투 로그는 7,200턴까지 풀 보존 (구버전 8시간 세이브 잔존 시에만 트림).

### 보물 굴림 메커니즘

`region.treasure.chance`는 **60초당 1회 독립 굴림 확률**로 해석. 즉 길이가 길수록 굴림 횟수가 비례 증가:

| 길이   | 굴림 횟수 | 평원(5%) 기대 적중 | 페이아웃 효율 (DISPATCH_REWARD_MULT) |
| ------ | --------: | -----------------: | -----------------------------------: |
| 60초   |         1 |             0.05회 |                                 1.00 |
| 1800초 |        30 |             1.50회 |                                 0.88 |
| 7200초 |       120 |             6.00회 |                                 0.75 |

페이아웃 자체에 `DISPATCH_REWARD_MULT`가 곱해지므로, **1초당 보물 골드 효율은 per-kill 효율 곡선과 동일한 1.00 : 0.88 : 0.75**. 활성 플레이 우대 + 방치 효율 동등 정체성을 보물에도 일관 적용. 검증: `scripts/treasure-rate-sim.ts`.

### 탐험 조기 종료

진행 중인 탐험을 "조기 종료" 버튼으로 중단할 수 있다. 부분 보상은 elapsedSec / playbackSec 비례:

- 적 처치분 자원·EXP·재료 → progress 비례 분배
- 보물 → `treasureRolls.slice(0, floor(elapsedSec / 60))`만 유효 (분당 굴림 자연 계승)
- finalMult는 **출발 시점에 선택한 길이의 효율**(1.0/0.88/0.75) 그대로 — 취소가 효율 우회 경로가 되지 않도록.
- 보스 탐험은 부분 보상 대상 외 (보상 0)
- 5초 미만 종료는 우발 클릭 보호로 보상 0

검증: `scripts/cancel-partial-sim.ts`.

## 필드 보스 (11종)

| 보스           | 지역        | 필드 보스 처치 시                          |
| -------------- | ----------- | ------------------------------------------ |
| 거대 슬라임 왕 | 평야        | 슬라임 왕의 핵 + 소환서 + (1% 슬라임 슈트) |
| 늑대 우두머리  | 외곽 숲     | 늑대 송곳니 + 소환서 + (1% 알파의 발톱)    |
| 거미 여왕      | 동굴        | 거미 여왕의 비단 + ...                     |
| 잠든 가디언    | 낡은 유적   | 잠든 가디언의 눈 + ...                     |
| 거대 전갈      | 사막        | 거대 전갈의 집게 + ...                     |
| 서리 거인      | 설원        | 서리 거인의 심장 + ...                     |
| 해적 선장      | 해적 정박지 | 선장의 해골 + ...                          |
| 유령 선장      | 유령선      | 유령선장의 모자 + ...                      |
| 심연 감시자    | 심연 입구   | shadow_fragment + ...                      |
| 균열의 수호자  | 심연 균열   | void_essence + ...                         |
| 심연의 군주    | 심연 핵     | 심연 군주의 핵 + ...                       |

각 보스 :

- 자체 스킬 (cooldown N턴)
- `drop`: 보장 재료
- `scrollDrop`: 소환서 (보장 + 확률)
- `uniqueDrop`: 1% 유니크 장비
- 쿨다운: `BOSS_COOLDOWN_MS = 60_000` (1분, 테스트값)

## 협동 보스 (Coop)

3종 (소환 필요):

| 보스   | 소환서                         | HP     | 지속 시간 |
| ------ | ------------------------------ | ------ | --------- |
| 산군   | 산군 소환서 (= 일반 소환서 ×5) | 15,000 | 2시간     |
| 그리폰 | 그리폰 소환서 (= ×10)          | 25,000 | 2시간     |
| 크라켄 | 크라켄 소환서 (= ×20)          | 50,000 | 2시간     |

특징:

- 서버 측에서 HP 공유, 여러 플레이어 협력
- 1회 공격 = 30턴 시뮬레이션 (`COOP_ATTACK_TURNS = 30`)
- 공격 쿨다운 30초 (`COOP_ATTACK_COOLDOWN_MS`)
- 처치 시 기여도에 따라 보상
- 미참여자는 보상 없음

## 대련 (Arena)

`src/app/ArenaPanel.tsx` + `src/lib/game/arena.ts`

### 구조

3티어 → 2티어 (NPC + 랜덤)로 단순화됨:

| 티어     | 상대                 | 설명        |
| -------- | -------------------- | ----------- |
| `npc`    | 정해진 NPC 시드      | 고정 난이도 |
| `random` | 등록된 다른 플레이어 | 변동 난이도 |

### 동작

- 다른 플레이어 닉네임으로 가져온 ArenaSnapshot을 적으로 사용
- `resolveBossDispatch`로 시뮬레이션 (보스 형태로 변환)
- 상대 정보는 `lastBattle.opponent*` 필드에 저장
- HP 회복은 별개로 처리 (`fightArena`)

## 전투 결과 표시

모든 전투는 `BattleLogViewer` 모달로 결과 확인 가능. `lastBattles[]`에 최근 3개 보관 (코옵은 별도로 `lastCoopBattles[]` 3개).
