## 🟠 High — 신뢰 경계·동기화 문제

### 1. 상점 구매가 전부 클라이언트 측에서 처리됨
- **범주**: Security / Bug
- **위치**: `src/app/page.tsx:502-531`, `src/adventure/ShopView.tsx`
- **문제**: 포션·재료·소비템 구매가 클라이언트에서 골드 차감/인벤토리 추가를 직접 수행. localStorage 변조 또는 응답 가로채기로 무한 구매 가능. 이후 마켓플레이스 등록·퀘스트 완료 등 서버 작업이 위조된 인벤토리를 신뢰함.
- **제안**: 구매 API를 신설해 서버에서 골드·인벤토리 캡·아이템 가용성 검증 후 결과를 반환.

### 2. 오프라인 시뮬레이션 충돌 시 보상이 조용히 사라짐
- **범주**: Sync Bug
- **위치**: `src/lib/storage/remote.ts:137-150`
- **문제**: 충돌 재시도 3회 초과 시 해당 키를 pending 큐에서 폐기. 오프라인 사냥 보상이 부분 동기화 직후 폐기되면 다음 로그인에서 그 결과가 영구히 누락된다.
- **제안**: stale 발생 시 전체 flush를 회복 큐에 적재하거나, 풀 싱크 성공 전까지 보상 적용을 보류.

### 3. 협동 보스 처치 처리에 CAS가 없어 중복 보상/방송 발생 가능
- **범주**: Race Condition
- **위치**: `src/app/api/coop/[region]/route.ts:294-305`
- **문제**: 동시 공격으로 두 명이 동시에 마지막 데미지를 넣으면 두 트랜잭션 모두 `defeated=true`로 판정. 스토리 플래그, 길드 알림, 채팅 방송, 리스폰 예약이 중복 실행.
- **제안**: HP 감소 직후 `WHERE defeated_at IS NULL` CAS, 또는 별도의 `defeated_by_user_id` 컬럼을 단일 원자적 UPDATE로 점유.

## 🟡 Medium — 게임 무결성·트랜잭션 결손

### 4. 재료 구매에 상한이 없음
- **범주**: Balance / Bug
- **위치**: `src/app/page.tsx:515-522`
- **문제**: 포션과 달리 `materialMax`가 없어 한 종류 재료를 무제한 구매 가능. 드롭 밸런스를 우회해 차익 거래·과잉 제작 가능.
- **제안**: 재료별/총량 상한 도입 후 구매 시점에 강제.

### 5. 인벤토리 가득 찼을 때 퀘스트 보상 재료가 조용히 소실
- **범주**: Bug / UX
- **위치**: `src/adventure/quests/applyReward.ts:83-85`, `src/app/page.tsx`
- **문제**: 포션은 오버플로우 표기가 있으나 재료는 캡 체크 없이 추가 시도. 토스트는 "+5 재료"라고 표시되지만 실제로는 2개만 들어감.
- **제안**: 포션과 같은 패턴으로 가용 공간 사전 검증 + 잉여 안내.

### 6. 마켓플레이스 인박스 클레임 시 아이템 추가 실패가 무시됨
- **범주**: State Consistency
- **위치**: `src/app/api/marketplace/inbox/claim/route.ts`
- **문제**: 인벤토리가 가득 차도 인박스 row는 claimed로 표시됨. 보상 영구 소실.
- **제안**: 클레임 시 공간 사전 검증, 부족 시 400 반환 또는 재클레임 가능 상태 유지.

### 7. 스토리 플래그 키가 서버/클라이언트에서 하드코딩 분기
- **범주**: Sync Bug
- **위치**: `src/app/api/coop/[region]/route.ts:466` vs `src/app/page.tsx:650`
- **문제**: 양쪽에 `storyFlags.v2`가 별도 리터럴로 박혀 있음. 이미 한 차례 운봉 지역 해금 버그 이력. 한쪽만 갱신되면 다시 깨진다.
- **제안**: 공유 모듈에서 enum/const로 단일 정의 후 import. 컴파일 타임 일치 보장.

### 8. 길드 퀘스트 완료 보상이 시점에 의존
- **범주**: Business Logic / Race
- **위치**: `src/app/api/guilds/quests/progress/route.ts:137-150`
- **문제**: 완료 순간 길드원 전원에게 inbox row insert. 직전 탈퇴자는 부당 수령, 직후 가입자는 누락. 멱등성 키 없음.
- **제안**: 퀘스트 시작 시점에 멤버 스냅샷을 저장하거나, 공유 풀에서 클레임 시점에 멤버십 검증.

### 9. 명성 기여에 시간 기반 레이트 리밋이 없음
- **범주**: Balance / Economy
- **위치**: `src/app/api/guilds/fame-contribute/route.ts:6-27`
- **문제**: 호출당 100 캡만 있어 짧은 시간에 다회 호출하면 부풀리기 가능. 다중 기기 시나리오에서 특히 취약.
- **제안**: 사용자별 시간 윈도우(예: 시간당/일당 상한) 추가. 마지막 기여 timestamp 추적.

### 10. 퀘스트 킬 타겟에 몬스터명 오타 검증 없음
- **범주**: Data Integrity
- **위치**: `src/adventure/quests/useQuests.ts:64-87`
- **문제**: `target.monsterName`이 `MONSTERS` 키와 일치하지 않으면 진행도가 영구 0. 무결성 테스트가 레시피/아이템만 검사함.
- **제안**: 무결성 테스트 확장 — `monsterName`, `materialId` 등 모든 참조 키를 자동 검증.

### 11. 마켓 리스팅에 아이템 스탯 스냅샷이 없음
- **범주**: Data Integrity / Audit
- **위치**: `src/app/api/marketplace/listings/route.ts:357-362`
- **문제**: `itemId`/`itemName` 문자열만 저장. 이후 아이템 밸런스 패치 시 거래 이력에서 기대치와 다른 결과 노출. 구매자가 실제로 무엇을 사는지 보장 못 함.
- **제안**: 스탯/희귀도/주요 보너스를 JSONB 스냅샷으로 함께 저장.

### 12. Clerk 사용자 라이프사이클이 명문화되어 있지 않음
- **범주**: Data Cleanup
- **위치**: `src/db/schema.ts:20-34`
- **문제**: 외부에서 Clerk 사용자 삭제 시 cascade로 게임 데이터가 즉시 소실. 재가입 시 새 ID로 새 캐릭터 시작 — 사용자 기대와 다를 수 있음.
- **제안**: 30일 grace period로 archive 후 hard delete. 정책 문서화.

## 🟡 Medium — 멀티탭/세션 일관성

### 13. 시련(trial) 상태가 탭 간 의도치 않게 소실
- **범주**: UX / State
- **위치**: `src/app/page.tsx:253-276`
- **문제**: 한 탭이 마을로 돌아가면 `endTrial`이 전역 localStorage를 비워 다른 탭의 진행 중 시련 상태가 사라짐. 영구 저장의 "이어하기" 계약 위반.
- **제안**: `BroadcastChannel`로 시련 종료를 트리거한 탭만 정리, 또는 세션 dedup 키로 격리.

### 14. HP 0 이하 보정 로직이 시점 의존
- **범주**: Bug
- **위치**: `src/app/page.tsx:391-409`
- **문제**: 백그라운드에서 HP 0으로 떨어진 후 복귀하면 `hp: 1` 보정이 단일 패스로만 실행. 특정 순서로 네비게이션 시 누락 가능.
- **제안**: `useLayoutEffect`로 매 렌더 가드, 또는 별도 정규화 단계 도입.

## 🟢 Low — 사용자 경험·관측성

### 15. 오프라인 시뮬레이션 보상 캡 도달 시 안내 없음
- **범주**: UX / Operational
- **위치**: `src/adventure/battle/offlineSim.ts:24,68-69`
- **문제**: 1시간(`OFFLINE_SIM_MAX_MS`) 초과 시 보상이 잘리지만 사용자에게 표시 없음. 장시간 자리비움 후 손해 인지 불가.
- **제안**: 캡 발생 시 토스트 안내 + 분석 로깅.

### 16. 마켓 TTL 값이 UI/문서에 노출되지 않음
- **범주**: UX
- **위치**: `src/app/api/marketplace/listings/route.ts:38,103`
- **문제**: 주석은 "24시간"이지만 상수 변경 시 UI 안내와 어긋날 수 있음.
- **제안**: 클라이언트에서 상수를 import해 표시. 만료 동작에 대한 픽스처 테스트 추가.

### 17. 상점 아이템 ID 존재 여부 무결성 검증 없음
- **범주**: Data Integrity
- **위치**: `src/adventure/ShopView.tsx`
- **문제**: 존재하지 않는 ID를 팔도록 설정해도 이름 폴백이 ID 문자열을 출력, 구매가 조용히 실패.
- **제안**: 무결성 테스트에 shop 카탈로그 추가, CI 실패 처리.

### 18. NPC 대사가 참조하는 스토리 플래그가 명세화 안 됨
- **범주**: Data Integrity
- **위치**: `src/adventure/data/npcs.ts`
- **문제**: 운향 등 후반 NPC 대사가 미설정 플래그를 전제하면 사용자가 맥락 없는 대사를 봄.
- **제안**: NPC별 의존 플래그를 데이터에 선언하고, 게임이 실제로 그 플래그를 세팅하는지 무결성 테스트로 검증.

### 19. 레시피 공유 배열 갱신이 read-modify-write
- **범주**: Race
- **위치**: `src/app/api/marketplace/listings/route.ts:304-306`
- **문제**: shareable 배열을 읽어 수정 후 저장. 동시 학습/공유 시 한쪽 갱신이 덮어쓰여 사라질 수 있음.
- **제안**: `onConflictDoUpdate`로 원자적 패치 또는 row-level lock.

### 20. 마켓 동시 구매 통합 테스트 없음
- **범주**: Test Gap
- **위치**: `src/app/api/marketplace/listings/buy/route.ts`
- **문제**: 조건부 UPDATE로 보호되어 있으나 동시성 시나리오 회귀 테스트가 없어 수정 시 사일런트 깨짐 가능.
- **제안**: 두 클라이언트가 동시에 buy 요청을 보냈을 때 정확히 하나만 성공, 다른 하나는 409인지 검증.

### 21. 오프라인 시뮬레이션 포션 픽업/리필 경로 미테스트
- **범주**: Test Gap
- **위치**: `src/adventure/battle/offlineSim.ts`
- **문제**: 포션 가득/소진 케이스와 도중 다른 종류 포션 선택 로직이 명시적 테스트로 보호되지 않음.
- **제안**: 픽업 로직을 함수로 추출하고 케이스별 단위 테스트.

### 22. 레벨 캡 클램프 검증 부족
- **범주**: Balance
- **위치**: `src/lib/leveling.ts`, `src/adventure/battle/offlineSim.ts`
- **문제**: 누적 EXP 적용 시 `MAX_LEVEL` 초과 EXP가 누적될 수 있음(시각적 차이 정도). 테스트로 보장돼 있지 않음.
- **제안**: `applyExpGain`에 레벨 클램프 명시 + 테스트.

### 23. 오프라인/실시간 전투 턴 계산 분기
- **범주**: Edge Case
- **위치**: `src/adventure/battle/offlineSim.ts:35-36`
- **문제**: 탭 비활성 시 `useBattle`은 멈추지만 오프라인 시뮬은 진행 — 동일 시간 동안 결과가 갈림. 가정이 문서화돼 있지 않음.
- **제안**: 분기 지점/가정을 코드 주석 또는 docs에 기록. 가능하면 턴 진행을 명시적 카운터로 통일.
