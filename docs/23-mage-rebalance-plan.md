## 0. 한 문장 요약

**1차 마법사는 기본 공격이 너무 약해 마법 스킬 쿨 사이를 못 메우고**, 2차 원소술사로 넘어가도 **선택지가 1개뿐**이라 빌드 다양성이 없다. 정체성 자원(MATK)은 격상되었지만(`matkToIntMult: 1.3`) `docs/20` 목표 1.4에 못 미치고, 마법 데미지를 증폭하는 세트/장비/크리 메커니즘이 없어 INT 성장의 보상이 작다.

## 2. 객관적 약세 근거 — `docs/19` 인용

### 2.1 1킬/턴 한계 지역 (1차 만렙 Lv100, 장비 0 가정)

`docs/19-kill-per-turn-by-class.md:124-138`:

| 직업   | 1킬/턴 가능 한계 | 격차                      |
| ------ | ---------------- | ------------------------- |
| 전사   | 동굴 (Lv 30~50)  | 베이스 ATK 312, DEF 두꺼움 |
| 도적   | 사막 (Lv 60~80)  | ATK 365 + 크리 15% (×2)   |
| **마법사** | **유적 빠듯**(Lv 30~50)   | ATK 211 (도적의 57%), DEF 50% 무시만 |

기본 공격으로 1킬/턴을 유지할 수 있는 지역이 **도적보다 2~3 단계 뒤떨어진다**. 마법 스킬을 쥐고 있어도 4~8턴 쿨에 묶여 있어 평균 DPS 보강이 제한적.

### 2.2 슬롯 경합 — 액티브 마법이 기본 공격을 잡아먹음

`logic.ts`의 페이즈 흐름은 SPD 기반 슬롯이 발동 가능한 스킬을 우선 소모. 마법사의 SPD=8 + AGI=3은 평균 슬롯 1~2개 수준이라:

- 마력 자동(4턴 쿨) 발동 턴 → 슬롯 1을 마법이 먹고 기본 공격 0회
- 비-마법 턴 → 약한 기본 공격 211 ATK가 적 DEF에 막힘

스킬 발동률은 1/4=25%. 그 중 절반은 단발(intMult 1.5×INT)이라 평균 DPS는 도적 대비 명백히 낮다.

### 2.3 마법 크리티컬 부재

`logic.ts`의 `magic_damage` effect 분기는 크리티컬 굴림 없음. 도적/어쌔신은 크리(×2 / ×2.5)로 폭발력을 가지지만 마법사는 항상 "정타". 이는 INT 성장이 크리 곱연산을 받지 못한다는 의미.

### 2.4 DOT 빌드 차단

`DOT_BASE_INT_MULT: 0.5`는 INT 기반 — 이론상 마법사가 가장 잘 운용해야 하지만 **마법사·원소술사 스킬 어느 것도 `apply_dot` 효과가 없다**. DOT는 맹독술사 전용. 결과: 마법사가 INT 빌드의 또 다른 출구(스택 누적·DOT 폭발)를 못 쓴다.

### 2.5 빌드 다양성: 2차 1개

`docs/17-second-job-quest-plan.md` 시련 매트릭스:
- 전사: 광전사 / 방패병 (2개)
- 도적: 어쌔신 / 맹독술사 (2개)
- **마법사: 원소술사 (1개)**

선택지 자체가 없어 후반 플레이가 단조롭다. 모든 마법사 캐릭터가 동일한 "마법 +60% / 턴 시작 자동 마법"으로 수렴.

### 2.6 장비/세트 — 마법 데미지 증폭 0개

전체 `EQUIPMENT` / `SET_BONUSES` 검토(Explore agent 보고서 §7):

- INT/MATK 부여 장비 ~5종 (정령의 지팡이, 망령의 로브 등) — 존재하지만 다른 직업의 ATK 부여 장비 수에 비해 절대 부족
- **세트 보너스 중 `magic_damage_amp` 항목 0개** — 물리 ATK 증폭 세트는 다수, 마법 증폭 세트는 전무

→ 원소술사의 매직 +60%가 패시브 외 유일한 마법 증폭 출처. 빌드 깊이 0.

## 4. 추천 패키지

### 4.1 최소 패키지 — "1차 진행 곡선만 보정" (1커밋, 코드 ~10줄)

**A1 + A2 + A3** 채택.

- A1: `matkToIntMult` 1.3→1.4 (정합성)
- A2: 마법 스킬 쿨 -1턴 일괄
- A3: `baseAtk` 12→14

**예상 효과**: 만렙 1차 마법사 평균 DPS +25~35%. 1킬/턴 한계가 "유적 빠듯" → "유적 안정 + 사막 진입 가능" 수준.

**리스크**: 광전사 출시 후 밸런스 곡선과 비교 시뮬 필요. `scripts/first-job-progression-sim.ts`의 mage 시뮬 결과로 검증.

### 4.2 균형 패키지 — "1차 + 2차 + 빌드 깊이" (2~3커밋, 코드 ~80줄)

**4.1 패키지 + B1 + B3** 추가.

- B1: 마법 크리 메커니즘 (INT×0.001, cap 25%, ×1.8) — 도적과의 폭발력 갭 절반 정도 메움
- B3: 마법 세트 보너스 1~2개 — 빌드 깊이 0→1 단계

**예상 효과**: 마법사 캐릭터에 "장비/세트 빌드"의 의미가 생김. 중후반 컨텐츠에서 도적·전사와 비등하게 경쟁.

**리스크**: B1 도입 시 보스 전투에서 마법사 폭발 턴이 보스 ATK 디버프 시점과 겹쳐 일격사가 늘어날 수 있음 — 보스 시뮬 필수.

### 4.3 풀 패키지 — "정체성 재구축" (장기 작업, 코드 200줄+)

**4.2 패키지 + B2 + C1** 추가.

- B2: DOT 채널 마법사에 개방 (맹독술사와 별 채널)
- C1: 2차 신직업(현자 또는 소환술사) + 시련 퀘스트 1세트

**예상 효과**: 마법사가 "정타 마법 / 크리 폭발 / DOT 지속"의 3 빌드를 가지는 깊은 직업이 됨. 전사·도적과 동등한 빌드 다양성.

**리스크**: 컨텐츠 작업량 큼(2주+). C1 신직업의 시련 디자인은 별도 plan 문서 필요.

## 5. 데이터 흐름 — B1(마법 크리) 상세 설계 예시

가장 도입 부담이 큰 항목이라 별도 §로 명세.

### 5.1 효과 모델

```ts
// types.ts — SkillEffect의 magic_damage variant에 옵션 추가
{ kind: "magic_damage", intMult: number,
  ignoreDef?: boolean, ignoreMdef?: boolean, hits?: number,
  // 신규 (모든 마법 데미지 자동 적용)
}

// logic.ts — magic_damage 분기 (대략)
const magicCritChance = Math.min(MAGIC_CRIT_CAP, stats.int * MAGIC_CRIT_RATE_PER_INT);
const isCrit = Math.random() < magicCritChance;
const critMult = isCrit ? MAGIC_CRIT_MULT : 1.0;
const dmg = floor((baseHit - effectiveMdef) * (1 + magicDmgAmpPct) * critMult);
```

### 5.2 상수 (data.ts에 추가)

```ts
export const MAGIC_CRIT_RATE_PER_INT = 0.001;  // INT 1당 0.1%
export const MAGIC_CRIT_CAP = 0.25;            // 최대 25%
export const MAGIC_CRIT_MULT = 1.8;            // ×1.8 (도적 ×2 보다 보수적)
```

### 5.3 만렙 마법사에서의 기대값

- INT ≈ 14 + 200 + 장비 ≈ 250~350
- 크리 확률 = 0.001 × 300 = 0.30 → cap 0.25 (25%)
- 평균 데미지 배수 = 1 + 0.25 × (1.8−1) = **1.20** (+20%)

### 5.4 도적과의 폭발력 비교

- 도적 크리: 15%×2 = +15% 평균 / 어쌔신: 50%×2.5 = +75%
- 마법사(B1 적용): 25%×1.8 = +20% 평균
- 보수적 설정으로 어쌔신을 따라잡진 않지만 도적과 동등 수준

### 5.5 보스 / DOT와의 상호작용

- 보스: 보스 ATK 디버프 턴에 마법사 크리 곱이 겹치면 ~20% 추가 → 시뮬에서 보스 클리어 턴이 1~2 빨라질 가능성. **수용 가능**.
- DOT: B2와 결합 시 DOT 데미지에는 크리 적용 안 함 (스택 누적 메커니즘이라 곱연산 부적합).

## 8. 리스크 / 회귀 가능성

- **광전사와의 균형**: 광전사 ATK +45%(`berserker_overdrive`)는 1차 베이스 위에 곱해진다. 마법사 baseAtk +2는 전사 영향 없음(다른 직업이라). 회귀 없음.
- **도적 크리(15% ×2)와의 정체성 충돌**: B1 도입 시 마법사 크리도 본질적으로 같은 메커니즘이지만 채널 다름(도적은 AGI, 마법사는 INT) — 직업 정체성 유지.
- **저장된 캐릭터 영향**: 모든 변경은 `computeStats`에서 매 호출 시 재계산되므로 로드 즉시 반영. 마이그레이션 불필요.
- **시뮬레이션 가정의 한계**: `scripts/first-job-progression-sim.ts`는 1턴 1킬 가정 모델. 마법 스킬의 다중 히트(연쇄 번개 ×3, 번개 사슬 ×4)는 단일 적에는 과잉이라 실제 실전 효율은 더 낮을 수 있음 — 시뮬 수치 그대로 믿지 말고 보스 시뮬도 함께 검토.
